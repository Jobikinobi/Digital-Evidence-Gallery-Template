## ğŸ¯ 4-Stage Workflow

### Overview
```
Your Evidence â†’ Stage 1 â†’ Stage 2 â†’ Stage 3 â†’ Professional Gallery
                 Vault     Optimize   Website
```

### Stage 1: Build Vault
```bash
# Add your evidence
cp /evidence/*.JPG 00-SOURCE-EVIDENCE/photos/
cp /evidence/*.pdf 00-SOURCE-EVIDENCE/documents/

# Build vault
python3 scripts/1_build_vault.py
```

**Creates**:
- Preserved originals in `01-EVIDENCE-VAULT/`
- EXIF metadata extraction
- SHA-256 checksums
- Vault manifest

### Stage 2: Optimize for Web
```bash
python3 scripts/2_optimize_for_web.py
```

**Creates**:
- Compressed photos in `02-WEB-OPTIMIZED/photos/`
- Thumbnails in `02-WEB-OPTIMIZED/thumbnails/`
- Rendered PDF pages in `02-WEB-OPTIMIZED/documents/`
- 85% file size reduction

### Stage 3: Generate Website
```bash
python3 scripts/3_generate_website.py
```

**Creates**:
- `03-WEBSITE-OUTPUT/index.html` (responsive gallery)
- Data files (images-data.json, documents-data.json)
- Complete metadata display
- Professional branding

### All Stages (One Command)
```bash
python3 scripts/run_all.py
```

**Total Time**: 10-30 minutes depending on evidence size

### What's Safe to Commit to Git

- Scripts (no evidence in them)
- Documentation
- .gitkeep files
- README files
- Configuration templates

âŒ **No** (Evidence data):

The included `.gitignore` automatically protects:
